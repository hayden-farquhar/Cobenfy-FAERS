# Real-World Pharmacovigilance of Xanomeline-Trospium (Cobenfy) in FDA FAERS

Reproducibility repository for the disproportionality analysis of xanomeline-trospium (Cobenfy/KarXT) adverse event reports in the FDA Adverse Event Reporting System (FAERS).

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.19623168.svg)](https://doi.org/10.5281/zenodo.19623168)

**Pre-registration:** [OSF doi:10.17605/OSF.IO/2PWUD](https://doi.org/10.17605/OSF.IO/2PWUD)

## Overview

This repository contains the complete analytical pipeline for a post-marketing pharmacovigilance analysis of xanomeline-trospium, the first non-dopamine D2 receptor antagonist antipsychotic (FDA-approved September 2024). The analysis applies a four-method disproportionality battery (ROR, PRR, IC, EBGM) with active-comparator analyses against six D2 antagonists, time-to-onset Weibull modelling, E-value sensitivity analysis, and multiple robustness checks to FAERS data spanning Q4 2024 through Q4 2025.

## Reproducing the Analysis

### Prerequisites

- Python 3.10+
- ~50 GB disk space for raw FAERS files
- ~1.5 GB for the DuckDB database

### Setup

```bash
pip install -r requirements.txt
```

### Running the Pipeline

Scripts are numbered and must be run sequentially:

```bash
# Step 1: Download FAERS quarterly ASCII files (Q4 2024 -- Q4 2025)
# WARNING: Downloads ~50 GB of data
python scripts/01_download_faers.py

# Step 2: Load into DuckDB and deduplicate
python scripts/02_load_deduplicate.py

# Step 3: Standardise drug names via RxNorm mapping
python scripts/03_drug_standardisation.py

# Step 4: Identify xanomeline-trospium and comparator cases
python scripts/04_case_identification.py

# Step 5: Primary disproportionality analysis (4-method battery)
python scripts/05_disproportionality.py

# Step 6: Active-comparator analysis vs 6 D2 antagonists
python scripts/06_active_comparator.py

# Step 7: Time-to-onset Weibull analysis
python scripts/07_time_to_onset.py

# Step 8: Sensitivity analyses (PS-only, US-only, HCP-only, Weber)
python scripts/08_sensitivity.py

# Step 9: Refit MGPS prior for EBGM correction
python scripts/09_mgps_refit.py

# Step 10: Supplementary analyses (outcome severity, indication-restricted, controls)
python scripts/10_supplementary.py

# Step 11: Additional validation (sex/age strata, E-values, CYP2D6, sequential, masking)
python scripts/11_additional_validation.py

# Step 12: Strengthening analyses (serious-only, polypharmacy, reporter-stratified, MH)
python scripts/12_strengthening.py

# Step 13: Generate publication figures
python scripts/13_figures.py

# Step 14: Generate case-selection flow diagram
python scripts/14_flow_diagram.py
```

Total runtime is approximately 30 minutes on a modern laptop (excluding the FAERS download in Step 1).

### Verifying Outputs

The `outputs/` directory contains the results generated by this pipeline. Running the scripts from scratch on the same FAERS data should reproduce these files exactly.

Key output files:
- `outputs/tables/disproportionality_cobenfy_full.csv` — full disproportionality results for 148 drug-PT pairs
- `outputs/tables/signals_cobenfy_consensus.csv` — 56 consensus signals (>=3/4 methods positive)
- `outputs/tables/active_comparator_results.csv` — 192 pairwise comparisons vs 6 comparators
- `outputs/supplementary/signal_classification_disease_vs_drug.csv` — pharmacological vs disease-manifestation classification
- `outputs/supplementary/e_values.csv` — E-values for unmeasured confounding robustness

## Data

Raw FAERS data files are not included in this repository due to size (~50 GB). See `data/README.md` for download instructions.

The intermediate DuckDB database (`data/processed/faers.duckdb`, ~1.4 GB) is also excluded. It is regenerated by scripts 01--03.

## Methods Summary

| Analysis | Method | Script |
|---|---|---|
| Signal detection | ROR, PRR, IC (BCPNN), EBGM (MGPS) | 05, 09 |
| Consensus threshold | >= 3 of 4 methods positive | 05 |
| Multiple testing | BH-FDR (ROR); Bayesian shrinkage (EBGM, IC) | 05, 09 |
| Active comparator | ROR with Bonferroni (alpha=0.05/192) | 06 |
| Demographic adjustment | Mantel-Haenszel (age x sex strata) | 12 |
| Time-to-onset | Weibull parameterisation | 07 |
| Confounding robustness | E-values (VanderWeele & Ding, 2017) | 11 |
| Signal classification | Clinical review: pharmacological vs disease manifestation | 12 |
| Sensitivity | PS-only, US-only, HCP-only, serious-only, polypharmacy-excluded | 08, 12 |
| Reporting bias | Weber effect, reporter-stratified ROR, sequential detection | 08, 11, 12 |

## Active Comparators

Six D2 antagonist antipsychotics were used as active comparators:
- Olanzapine (n=6,926)
- Risperidone (n=6,254)
- Aripiprazole (n=5,516)
- Quetiapine (n=9,074)
- Lurasidone (n=806)
- Brexpiprazole (n=1,288)

## Key Findings

- **1,423** xanomeline-trospium suspect cases identified across 5 FAERS quarters
- **56** consensus disproportionality signals: 37 pharmacological, 19 disease manifestations
- Gastrointestinal (nausea ROR 7.5, vomiting 8.4) and muscarinic/anticholinergic (urinary retention ROR 44.0, drooling 61.0) signals dominant
- Weight gain, EPS, metabolic events, tardive dyskinesia, and hyperprolactinaemia absent or significantly reduced vs all 6 D2 antagonist comparators
- All major adverse events early-onset (median 0--3 days, Weibull beta < 1)
- 46 of 56 signals robust to unmeasured confounding (E-value lower CI >= 3.0)
- Drooling signal persists after excluding clozapine co-prescription (ROR 61.0 to 60.0)
